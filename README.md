# 📺 IPTV Manager — Full Stack Render Deployment

Pure 302-redirect IPTV playlist manager. No stream proxying — server just redirects players directly to source URLs. DRM channels are automatically stripped.

---

## 🚀 Deploy to Render (One-Time Setup)

### 1. Push to GitHub
```bash
git add .
git commit -m "deploy"
git push origin main
```

### 2. Create Web Service on Render
1. Go to [render.com](https://render.com) → **New → Web Service**
2. Connect your GitHub repo
3. Render auto-detects `render.yaml` → uses Docker runtime
4. Click **Deploy**

### 3. First Deploy Time
| Stage | Time |
|---|---|
| Install deps (cached after first run) | ~2 min |
| Vite build (cached when src unchanged) | ~1 min |
| Docker image transfer | ~30s |
| Server startup | ~2s |
| **Total cold start** | ~3-4 min |
| **Subsequent deploys (code change only)** | ~45-90s |

### 4. Sync Your Data
1. Open your Render URL (e.g. `https://iptv-manager.onrender.com`)
2. Go to **Server** tab → enter your URL → click **☁️ Sync to Server**
3. Done!

| Kodi mode | `?kodi=1` appended to any playlist URL |

## 🔀 Stream Routing

```
Player requests /redirect/:id
        │
        └─► 302 redirect → original stream URL
                │
                └─► Player connects directly to source (server not involved)
```

For multi-source channels (same channel in multiple sources):
```
Player requests /redirect/best/:channelName
        │
        └─► Server HEAD-checks all source URLs in parallel
                │
                └─► 302 redirect to fastest live link
```

| Endpoint | Description |
|---|---|
| `GET /health` | Server status |

| `POST /api/health/batch` | Batch check (body: `{ids: [...]}`) |
| `GET /api/bestlink/:name` | All links for channel, ranked by latency |

## 💓 Free Tier Keepalive

Server automatically pings `/health` every **14 minutes** to prevent Render free tier from sleeping. No configuration needed.

## 📁 Architecture

```
server.cjs          ← Express 4, CommonJS, pure 302 redirect
dist/               ← Built React frontend (served as static files)
/data/db/db.json    ← Persistent channel/source/playlist data (Render disk)
```

```
Dockerfile:
  Stage 1: deps    → npm install (cached)
  Stage 2: builder → vite build  (cached)
  Stage 3: runner  → node:20-alpine + curl only (~120MB)
```
